**Descrição:** Página de desapego do Renato — vende móveis e eletros usados, cada item com botão "dar um lance" que abre o WhatsApp pré-preenchido. Sem banco de dados; todo o conteúdo (incluindo fotos em base64) vive dentro do `docs/index.html`.

> **Arquitetura de hospedagem:** o site público fica em `docs/index.html`. Tanto o GitHub
> Pages (source = branch `main` / `/docs`) quanto o Cloudflare Workers (`wrangler.jsonc`,
> `assets.directory = "./docs"`) servem essa pasta. Servir de `docs/` garante que arquivos
> internos (CLAUDE.md, memórias, .git) NÃO sejam expostos no site.

## Modelo de Dados (dentro do docs/index.html)

```js
const OWNER = { name: "Renato", whatsapp: "5519996614321" }; // 55 + DDD 19 + número
const ITEMS = [ { id, title, description, pricePaid, askingPrice, refs[], photos[], sold }, ... ];
```

- `pricePaid` = preço "de" (quanto custou). `askingPrice` = preço "por" (quanto quer).
- `sold: true` → item fica cinza, tachado e sem botão de lance.
- "dar um lance" → abre `wa.me` do OWNER com mensagem pré-preenchida.

## Comandos Úteis

```bash
# Rodar localmente
cd docs && python3 -m http.server 8000   # http://localhost:8000

# Publicar mudança (Pages + Cloudflare atualizam sozinhos em ~1min)
git add docs/index.html && git commit -m "atualiza itens" && git push

# Deploy manual no Cloudflare (se o build automático falhar)
npx wrangler login   # 1a vez / token expirado
npx wrangler deploy  # -> https://desapego.renattowolf.workers.dev/
```
